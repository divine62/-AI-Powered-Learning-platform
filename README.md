## Overview

**Knowtify** is a full-stack AI-powered learning platform designed specifically for Indian engineering students preparing for university exams. The core differentiator is **syllabus-aware YouTube video recommendations** — students upload their university syllabus, specify how many days remain before their exam, and receive ranked YouTube content matched against extracted syllabus topics.

Beyond video discovery, Knowtify transforms passive video watching into active learning through AI-generated notes (grounded in video transcripts), auto-generated quizzes, and a persistent AI tutor with session history.

---

## Features

### 🎯 Syllabus-Aware Video Finder
- Upload a university syllabus (PDF or image)
- PyMuPDF + Tesseract OCR extracts topics from scanned or text-based PDFs
- TF-IDF ranking with **synonym expansion** (e.g., "BST" → "binary search tree") surfaces the most relevant YouTube videos
- Videos matched per topic with duration filtering

### 📝 AI Note Generation
- Fetches the full YouTube transcript for any video
- Gemini API generates structured, transcript-grounded notes
- Notes are scoped to actual video content — no hallucinated information

### 🧠 AI Quiz Generation
- Per-video quizzes generated by Gemini
- JWT-authenticated so quizzes are tied to user accounts
- Multiple-choice format with instant feedback

### 🤖 AI Tutor (Persistent Chat)
- Full multi-turn chat at `/chat`
- SQLite-backed session history with automatic session titling
- Proactive topic suggestions returned from the backend
- Conversation context preserved across sessions

### 📊 User Dashboard
- Streak tracking and watch history
- Personalized recommendations based on past activity
- Profile management

### 🔐 Authentication
- JWT-based auth (access + refresh tokens)
- Secure registration and login
- Protected routes on both frontend and backend

## How It Works

### TF-IDF Ranking with Synonym Expansion

The core NLP pipeline handles vocabulary mismatch between syllabus language and YouTube video titles/descriptions:

1. Topics are extracted from the uploaded syllabus via PyMuPDF or Tesseract OCR
2. Each topic is expanded with domain synonyms (e.g., "DS" → "data structures", "OS" → "operating system")
3. YouTube Data API v3 returns candidate videos for each topic
4. TF-IDF vectorization scores each video's title + description against the expanded topic query
5. Videos are ranked by cosine similarity score and returned to the frontend

### Gemini Multi-Model Fallback

Because Gemini model names deprecate frequently, the backend implements a retry system across three models in order of preference:

```
gemini-2.5-flash → gemini-2.0-flash → gemini-2.0-flash-lite
```

Each call includes exponential backoff to handle transient API errors gracefully.

### PDF Syllabus Parsing

- Text-based PDFs: PyMuPDF extracts text directly
- Scanned/image PDFs: Pages are rasterized and passed through Tesseract OCR
- Extracted text is chunked into topic units for TF-IDF processing

## Getting Started

### Prerequisites

- Python 3.10+
- Node.js 18+
- Tesseract OCR installed and added to PATH
  - Windows: [UB Mannheim installer](https://github.com/UB-Mannheim/tesseract/wiki)
  - Linux: `sudo apt install tesseract-ocr`
- Google Gemini API key
- YouTube Data API v3 key

### Backend Setup

```bash
# Clone the repository
git clone https://github.com/your-username/knowtify-backend.git
cd knowtify-backend

# Create and activate virtual environment
python -m venv venv
source venv/bin/activate        # Linux/macOS
venv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt

# Create .env file (see Environment Variables section)
cp .env.example .env

# Run the development server
python app.py
# Flask runs on http://localhost:5000
```

### Frontend Setup

```bash
# Clone the repository
git clone https://github.com/your-username/knowtify-frontend.git
cd knowtify-frontend/knowtify

# Install dependencies
npm install

# Create .env file
cp .env.example .env
# Set REACT_APP_API_URL=http://localhost:5000

# Start the development server
npm start
# React runs on http://localhost:3000
```

## Deployment

### Backend — Cloudflare / Render (Docker)

```dockerfile
FROM python:3.11-slim

# Install Tesseract at OS level
RUN apt-get update && apt-get install -y \
    tesseract-ocr \
    libglib2.0-0 \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

# Dynamic port for cloud platforms
CMD gunicorn app:app --bind 0.0.0.0:$PORT --timeout 120 --workers 2
```

### Frontend — Vercel

```bash
# In Vercel dashboard or CLI:
# Build Command: npm run build
# Output Directory: build
# Environment Variable: REACT_APP_API_URL=<your-backend-url>
```

> **Note on SQLite:** The current deployment uses SQLite with an ephemeral disk. Data is reset on redeployment. For production, migrate to PostgreSQL.

## Known Limitations

- **SQLite + ephemeral disk:** Data does not persist across redeployments on platforms with ephemeral storage. Planned migration to PostgreSQL for production.
- **YouTube Transcript API:** Not all YouTube videos have captions available. Note generation gracefully fails when transcripts are unavailable.
- **Tesseract OCR accuracy:** Heavily formatted or low-resolution syllabus scans may yield imperfect topic extraction.
- **Duration filtering:** The video duration cap returns an empty result set if no videos match — there is no fallback to a wider filter.
